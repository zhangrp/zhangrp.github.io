Java线程基础

Daemon线程

守护线程是程序运行时在后台提供服务的线程，不属于程序中不可或缺的部分。 当所有非守护线程结束时，程序也就终止，同时会杀死所有守护线程。main() 属于非守护线程。使用 setDaemon() 方法将一个线程设置为守护线程。

守护线程的典型场景就是：垃圾回收线程

public static void main(String[] args) {
    Thread thread = new Thread(new MyRunnable());
    thread.setDaemon(true);
}

sleep()

Thread.sleep(millisec) 方法会休眠当前正在执行的线程，millisec 单位为毫秒。sleep() 可能会抛出 InterruptedException，因为异常不能跨线程传播回 main() 中，因此必须在本地进行处理。线程中抛出的其它异常也同样需要在本地进行处理。

yield()

对静态方法 Thread.yield() 的调用声明了当前线程已经完成了生命周期中最重要的部分，可以切换给其它线程来执行。该方法只是对线程调度器的一个建议，而且也只是建议具有相同优先级的其它线程可以运行。

public static native void yield();

join()

来自Thread类型，有三种重载形式，最终调用要么是wait(0)无限等待，要么是调用wait(delay)超时等待，非静态方法。

public final synchronized void join(long millis) throws InterruptedException {
    long base = System.currentTimeMillis();
    long now = 0;
    if (millis < 0) {
        throw new IllegalArgumentException("timeout value is negative");
    }
    if (millis == 0) {
        while (isAlive()) {
            wait(0);
        }
    } else {
        while (isAlive()) {
            long delay = millis - now;
            if (delay <= 0) {
                break;
            }
            wait(delay);
            now = System.currentTimeMillis() - base;
        }
    }
}

public final synchronized void join(long millis, int nanos)throws InterruptedException {
    if (millis < 0) {
        throw new IllegalArgumentException("timeout value is negative");
    }
    if (nanos < 0 || nanos > 999999) {
        throw new IllegalArgumentException(
                            "nanosecond timeout value out of range");
    }
    if (nanos >= 500000 || (nanos != 0 && millis == 0)) {
        millis++;
    }
    join(millis);
}

public final void join() throws InterruptedException {
    join(0);
}

在线程中调用另一个线程的 join() 方法，会将当前线程挂起，而不是忙等待，直到目标线程结束。执行thread.join()的线程等待thread线程的结束后再往下执行。

wait() notify() notifyAll()

wait和notify都是来之Object类，非静态的。

public final native void wait(long timeout) throws InterruptedException;
public final void wait(long timeout, int nanos) throws InterruptedException {
    if (timeout < 0) {
        throw new IllegalArgumentException("timeout value is negative");
    }
    if (nanos < 0 || nanos > 999999) {
        throw new IllegalArgumentException(
                            "nanosecond timeout value out of range");
    }
    if (nanos >= 500000 || (nanos != 0 && timeout == 0)) {
        timeout++;
    }
    wait(timeout);
}
public final void wait() throws InterruptedException {
    wait(0);
}

public final native void notify();
public final native void notifyAll();

调用 wait() 使得线程等待某个条件满足，线程在等待时会被挂起，当其他线程的运行使得这个条件满足时，其它线程会调用 notify() 或者 notifyAll() 来唤醒挂起的线程。它们都属于 Object 的一部分，而不属于 Thread。 只能用在同步方法或者同步控制块中使用，否则会在运行时抛出 IllegalMonitorStateException。

注意：
1、需要使用while检查条件并wait，而不是用if
2、使用notifyAll而不是notify，因为notify唤醒的是一个，比如生产者可能又唤醒了一个生产者，而不是唤醒一个消费者
3、使用 wait() 挂起期间，线程会释放锁。这是因为，如果没有释放锁，那么其它线程就无法进入对象的同步方法或者同步控制块中，那么就无法执行notify()或者notifyAll()来唤醒挂起的线程，造成死锁
4、notify必须在wait之后执行，否者将无法唤醒，而LockSupport.unpark(trhead)不是必须在LockSupport.park()之前，它内部是通过状态为来标示的。

LockSupport

@Test
public void testLockSupport() {

    Thread thread1 = new Thread() {
        @Override
        public void run() {
            LockSupport.park();
            System.out.println("thread2 in ");
            System.out.println("thread2 sleep.... ");
            try {
                Thread.sleep(10 * 1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println("thread2 end ");
        }
    };

    thread1.start();

    try {
        Thread.sleep(3 * 1000);
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
    System.out.println("main sleep 3s");
    LockSupport.unpark(thread1);

    try {
        thread1.join();
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
}

1、通过二元信号量实现阻塞，信号量最多只能加到1
2、底层是调用了Unsafe类的park方法，允许设置一个blocker对象，主要用来给监视工具和诊断工具确定线程受阻塞的原因
3、休眠时线程状态为WAITING
4、中断时不需要捕获InterruptedException异常
5、不会释放锁
6、通过unpark唤醒，unpark方法可以比park先执行，不会丢失唤醒信号

await() signal() signalAll()

java.util.concurrent类库中提供了Condition类来实现线程之间的协调，可以在Condition上调用await()方法使线程等待，其它线程调用signal()或signalAll()方法唤醒等待的线程。

相比于wait()这种等待方式，await()可以指定等待的条件，因此更加灵活。 使用Lock来获取一个Condition对象。参数下面的生成者和消费者。

public class ProduceAndConsumer {
    ReentrantLock reentrantLock = new ReentrantLock();
    Condition notEmpty = reentrantLock.newCondition();
    Condition notFull = reentrantLock.newCondition();

    public List<Integer> factory = new ArrayList<Integer>();
    public Integer size = 0;
    public static Integer max_size = 10;

    public void produce(Integer i) {
        reentrantLock.lock();
        try {
            while (size >= 10) { //注意这个地方一定要while，和wait，notify一样
                try {
                    notFull.await();
                } catch (Exception e) {

                }
            }
            factory.add(i);
            size++;
            notEmpty.signalAll();
        } finally {
            reentrantLock.unlock();
        }
    }

    public void consumer() {
        reentrantLock.lock();
        try {
            while (size == 0) {
                try {
                    notEmpty.await();
                } catch (Exception e) {

                }
            }
            System.out.println(factory.remove(0));
            size--;
            notFull.signalAll();
        } finally {
            reentrantLock.unlock();
        }
    }
}
