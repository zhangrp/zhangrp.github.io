选择排序

每一趟从待排序的数据元素中选出最小（或最大）的一个元素，顺序放在已排好序的数列的最前（最后），直到所有待排序的数据元素排完。

public void sort(int a[]) {
    if(a == null || a.length == 0 || a.length == 1){
        return;
    }
    for(int i=0;i<a.length;i++){
        int k = i;
        for(int j=i+1;j<a.length;j++){
            if(a[j] < a[k]){
                k = j;
            }
        }
        if(k != i){
            swap(a, i, k);
        }
    }
}
private void swap(int a[], int i, int j){
    int p = a[i];
    a[i] = a[j];
    a[j] = p;
}
  • 时间复杂度分析：O(n^2)

为什么时间复杂度是O(n^2)？
假如当n个元素时时间复杂度是f(n)，可以得出f(n) = f(n-1)+n。类推，f(n-1)=f(n-2)+n-1, f(n-2)=f(n-3)+n-2.... f(n)=n+(n-1)+(n-2)+(n-3)+...+1=n(n-1)/2。当n趋近于无穷大时f(n)趋近与n^2。

  • 不稳定排序

在一趟选择，如果当前元素比一个元素小，而该小的元素又出现在一个和当前元素相等的元素后面，那么交换后稳定性就被破坏了。比如：5 8 5 2 9，第一趟排序5就和2交换了，稳定性已破坏。 最坏，最好和平均情况下时间复杂度均为O(n^2)
